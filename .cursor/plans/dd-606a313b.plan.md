# Fix DDP logging and validation reporting

目标：

- 在分布式训练中只让 `rank 0` 输出训练/通用日志，避免重复；
- 在验证/测试阶段，把每个进程的局部分数收集到 `rank 0`，先打印各 rank 的分数，再打印汇总平均值（保留原有的 TensorBoard/WANDB 记录）。

要修改的文件：

- `main_train_vrt.py` — 训练循环中日志与验证汇总（guard logger calls; add per-rank gather-and-print）
- `main_test_vrt.py` — 测试脚本的汇总打印（收集每个 rank 的本地平均并在 rank 0 打印）
- `utils/utils_logger.py` — 在 logger 初始化时确保不会为非主进程附加 stream/file handler（或根据 `utils_dist.is_main_process()` 控制），以避免多进程重复输出。

主要实现要点（概要代码片段，供实现参考）：

- 在日志处统一使用分布式判断：
```python
from utils.utils_dist import is_main_process, get_rank, get_world_size
# only log on master
if is_main_process():
    logger.info('...')
```

- 验证/测试时收集每个 rank 的局部统计（使用 `all_gather` 或 `all_reduce` + per-rank tensors），在 `rank 0` 上打印每个 rank 的平均，然后打印全局平均：
```python
local = torch.tensor([local_psnr_sum, local_count], dtype=torch.float64, device=device)
all_local = [torch.zeros_like(local) for _ in range(world_size)]
dist.all_gather(all_local, local)
if is_main_process():
    for r, t in enumerate(all_local):
        psnr_sum, cnt = t.tolist()
        ave = psnr_sum / max(int(round(cnt)), 1)
        logger.info(f'[Rank {r}] Average PSNR: {ave:.2f} dB')
    # then compute global avg and logger.info once
```


验收标准：

- 训练期间仅 `rank 0` 输出训练日志（loss、timings、保存模型信息等）；
- 验证/测试阶段 `rank 0` 上能看到每个 GPU/rank 的单独平均分数行，以及最终的全局平均值；
- TensorBoard/WANDB 记录保持不变（仍由 `tb_logger` 在主进程写入）。

风险与注意事项：

- 若代码中存在直接 `print()` 而非 `logger.info()` 的位置，需要替换或加 `is_main_process()` guard；我们会在实现中查找并修正这些点。
- 使用 `dist.all_gather` 需要 `torch.distributed` 已初始化；代码应先判定 `is_dist_available_and_initialized()`。
