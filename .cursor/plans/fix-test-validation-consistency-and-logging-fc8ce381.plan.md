# 修复测试代码与验证一致性问题及日志输出问题

## 问题分析

### 问题1: 测试代码与验证代码不一致导致张量形状错误

- 验证代码（`models/model_vrt.py`）使用 `_assert_lq_channels` 进行通道验证
- 测试代码（`main_test_vrt.py`）缺少通道验证
- 验证代码的填充逻辑与测试代码不同
- 验证代码使用 `self.opt['netG'].get('out_chans', c)` 获取输出通道，测试代码可能未正确获取
- 验证代码的 `overlap_size` 处理与测试代码不同

### 问题2: 日志限制导致其他进程的测试结果无法输出

- `main_test_vrt.py` 中所有 print 语句都被 `args.rank == 0` 限制
- 在分布式测试中，每个 GPU 处理不同的视频片段，但只有 rank 0 输出日志
- 需要允许所有进程输出其处理的测试结果

## 修复方案

### 1. 对齐测试代码与验证代码的逻辑 (`main_test_vrt.py`)

#### 1.1 添加通道验证函数

- 在 `main_test_vrt.py` 中添加类似 `_assert_lq_channels` 的验证函数
- 在 `test_video` 和 `test_clip` 函数开始时进行通道验证

#### 1.2 修复 `test_video` 函数

- 对齐填充逻辑：使用与验证代码相同的填充计算方式
- 确保输出通道数计算与验证代码一致
- 使用 `args.netG_cfg.get('out_chans', c)` 获取输出通道数

#### 1.3 修复 `test_clip` 函数

- 对齐填充逻辑：使用与验证代码相同的填充计算方式
- 确保 `overlap_size` 的处理与验证代码一致（验证代码使用硬编码的 20，但测试代码应该从配置读取）
- 确保模型输出处理与验证代码一致

### 2. 修复日志输出问题 (`main_test_vrt.py`)

#### 2.1 移除或修改 rank 限制

- 移除测试过程中的 `args.rank == 0` 限制，允许所有进程输出日志
- 对于每个 clip 的测试结果，添加 rank 信息以便区分
- 对于最终汇总结果，保持只在 rank 0 输出

#### 2.2 改进日志格式

- 在日志中包含 rank 信息：`[Rank {rank}] Testing ...`
- 确保每个进程的测试结果都能被记录
