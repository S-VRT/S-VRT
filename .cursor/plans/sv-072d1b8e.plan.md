# S-VRT 全量重构（src 内新工程）

## 目标

- 在 `src/` 内重建完整训练/验证/推理工程，不再调用旧代码；外部旧文件不再使用。
- 全新 YAML/py 配置，覆盖旧 JSON 的全部配置项（等价语义，字段映射完整）。
- 保留分布式、日志、数据准备、LMDB、评测等原有能力，接口更简洁可插拔。

## 计划

1) 配置体系重建

- 定义 `configs/` YAML：拆分 base / dataset / model / train / eval；提供字段映射表覆盖旧 JSON。 
- 新 `src/config/parser.py`：支持分层合并、CLI override、DDP 环境注入。

2) 核心基础层

- 在 `src/core/` 实现 Registry/Builder（模型/数据/优化器/调度器/损失），分布式/随机种子/日志抽象，I/O 与路径管理。
- 统一日志：控制台+文件，选配 TensorBoard/W&B，保持原有日志信息粒度。

3) 数据层

- 迁移并重写数据准备与加载：Spike/RGB 数据集、meta 解析、LMDB 支持、缓存/多线程；拆分 transforms/collate。
- 数据准备工具 `src/tools/prepare_data.py`：包含原脚本功能（下载/校验/生成 LMDB/统计）。

4) 模型层

- 将 VRT 及子模块重构到 `src/models/vrt/`（blocks, model）；支持多任务配置（去模糊/超分/去噪）；注册到 Registry。
- 损失与正则化：重写到 `src/models/losses/` 并在 Builder 注册；保持 Charbonnier/SSIM/orthogonal 等能力。

5) 训练/验证运行层

- `src/engine/runner.py`：迭代/epoch 训练循环，支持 AMP、DDP、梯度裁剪、EMA、梯度累积、断点恢复。
- Hooks：日志、lr 调度、评测、checkpoint、EMA、profiling，对齐旧行为。
- Metrics：PSNR/SSIM/VFI 指标；评测数据管线。

6) 推理与导出

- `src/inference/`：批量/单视频推理，tile 测试，输出保存；支持模型导出（TorchScript/ONNX stub）。

7) 入口脚本与可执行

- 新 `train.py` / `test.py` / `infer.py` / `prepare_data.py` 仅依赖 `src/`，不引用旧代码。
- CLI 参数映射到 YAML，支持 torchrun 启动。

8) 清理与验证

- 不再使用仓库根旧代码；在 README 追加迁移指引与字段映射表。
- 添加冒烟测试：config 解析、单 step 训练、DataLoader 可迭代、模型前向、推理一帧。
